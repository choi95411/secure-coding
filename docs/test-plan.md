# Test Plan

계층은 모델 제약, Form/서비스 단위, View·DB 통합, 인증·권한, 파일 업로드, WebSocket, PostgreSQL 트랜잭션·동시성, Playwright E2E다.

작업 1은 가입·중복·비밀번호 해시·차단 로그인·POST 로그아웃·마이페이지 인증·본인 프로필·XSS, 상품 등록·가짜 이미지·비로그인·소유권·소프트 삭제·비공개/차단/삭제 미노출, 검색 제목/설명·상태·정렬·빈 값·SQLi 문자열·Reflected XSS를 검증한다.

후속 E2E는 A/B 가입 → A 상품 등록 → B 검색·상세 → 1대1 채팅 → B가 A에게 포인트 송금 → 잔액/원장 → 신고 → 관리자 제재 → 일반 사용자 관리자 접근 차단 순서다. 송금 경쟁 테스트는 PostgreSQL의 별도 트랜잭션 연결로 수행한다.
